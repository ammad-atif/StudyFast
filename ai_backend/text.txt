In java can we create a child object in parent pointer

Yes. In Java, a parent class reference can point to a child class object. This is called upcasting and is one of the core concepts of polymorphism.

Example:

Java
class Animal {
    void sound() {
        System.out.println("Animal sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Bark");
    }

    void run() {
        System.out.println("Dog runs");
    }
}

public class Main {
    public static void main(String[] args) {

        Animal a = new Dog(); // Parent reference, child object

        a.sound(); // Bark
    }
}

Here:

Animal → parent class

Dog → child class

a is a parent pointer/reference

new Dog() creates a child object

So this is completely valid:

Java
Animal a = new Dog();

Important points:

You can access only the methods available in the parent reference directly.

Java
a.sound(); // allowed
a.run();   // error

Even though the object is Dog, the reference type is Animal.

Overridden methods use the child version at runtime.

Java
a.sound();

calls Dog's sound() because of runtime polymorphism.

To access child-specific methods, you need downcasting:

Java
Dog d = (Dog) a;
d.run();

This concept is heavily used in:

collections

frameworks

interfaces

dependency injection

Spring

Android

game engines

design patterns like Factory and Strategy
